# üéì SynClass

**Sistem Informasi Akademik Terintegrasi**

SynClass adalah aplikasi web modern berbasis Next.js yang dirancang untuk mengelola kegiatan akademik perguruan tinggi secara efisien. Aplikasi ini menyediakan antarmuka yang intuitif untuk mahasiswa, dosen, dan kaprodi dalam mengelola data akademik, kelas, nilai, dan informasi pengguna.

## ‚ú® Fitur Utama

### üè† Dashboard Dinamis

Dashboard yang berbeda untuk setiap role pengguna dengan data real-time dari database.

#### Dashboard Mahasiswa

| Fitur                 | Deskripsi                                                 |
| --------------------- | --------------------------------------------------------- |
| **Stats Cards**       | Total SKS, Persentase Kehadiran, Progress Tugas, IPK      |
| **Jadwal Hari Ini**   | Daftar kelas yang harus diikuti hari ini                  |
| **Deadline Tugas**    | Tugas yang akan segera deadline dengan status pengumpulan |
| **Aktivitas Terbaru** | Riwayat presensi dan pengumpulan tugas                    |
| **Status KRS**        | Badge status pengajuan KRS (Pending/Approved/Rejected)    |
| **Tips Dinamis**      | Saran berdasarkan kondisi akademik                        |

#### Dashboard Dosen

| Fitur                   | Deskripsi                                                 |
| ----------------------- | --------------------------------------------------------- |
| **Stats Cards**         | Mata Kuliah Diampu, Total Mahasiswa, Sesi Presensi, Tugas |
| **Jadwal Mengajar**     | Kelas yang harus diajar hari ini dengan jumlah mahasiswa  |
| **Aktivitas Terbaru**   | Submission tugas mahasiswa, presensi yang dibuat          |
| **Postingan Terbaru**   | Materi, tugas, pengumuman yang telah dibuat               |
| **Indikator Penilaian** | Jumlah tugas yang belum dinilai                           |

#### Dashboard Kaprodi

| Fitur                | Deskripsi                                              |
| -------------------- | ------------------------------------------------------ |
| **Stats Cards**      | Total Mahasiswa, Total Dosen, Mata Kuliah, KRS Pending |
| **KRS Summary**      | Ringkasan status KRS (Approved, Pending, Rejected)     |
| **KRS Pending List** | Daftar pengajuan KRS yang menunggu persetujuan         |
| **Pengguna Terbaru** | User yang baru terdaftar di sistem                     |
| **Akses Cepat**      | Navigasi ke fitur manajemen                            |

### üîê Autentikasi & Keamanan

- **Login Terintegrasi** - Supabase Auth dengan session management
- **Role-Based Access** - Kontrol akses berdasarkan role (student/dosen/kaprodi)
- **Admin API** - Endpoint khusus untuk operasi admin dengan service_role key
- **Logout Aman** - Pembersihan sesi dan localStorage

### üìö Manajemen Akademik

#### Kartu Rencana Studi (KRS)

- **Mahasiswa**: Pilih mata kuliah, ajukan KRS, lihat status
- **Kaprodi**: Review, approve/reject pengajuan dengan catatan

#### Kartu Hasil Studi (KHS)

- Nilai per mata kuliah dengan grade huruf
- Perhitungan IPS dan IPK otomatis
- Riwayat akademik per semester

#### List Kelas

- Daftar kelas yang diikuti/diampu
- Informasi jadwal, ruangan, dosen
- Akses ke detail kelas

### ‚öôÔ∏è Pengaturan Sistem

#### Manajemen User (Kaprodi Only)

- CRUD pengguna dengan Admin API
- Filter berdasarkan role
- Form lengkap: Email, Password, Username, NIM/NIDN, Jurusan, Fakultas, Angkatan

#### Manajemen Mata Kuliah

- CRUD mata kuliah
- Pengelompokan per semester
- Pencarian dan statistik

## üóÑ Konfigurasi Database

### Tabel Database

```sql
-- 1. Tabel Profiles (User Data)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  email TEXT,
  username TEXT,
  role TEXT DEFAULT 'student' CHECK (role IN ('student', 'dosen', 'kaprodi')),
  nim BIGINT,                    -- Untuk mahasiswa
  nidn TEXT,                     -- Untuk dosen/kaprodi
  jurusan TEXT,
  fakultas TEXT,
  angkatan TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 2. Tabel Mata Kuliah
CREATE TABLE matakuliah (
  id SERIAL PRIMARY KEY,
  kode_mk TEXT NOT NULL,
  nama_mk TEXT NOT NULL,
  sks INTEGER NOT NULL,
  semester INTEGER NOT NULL,
  hari TEXT,
  jam_mulai TIME,
  jam_selesai TIME,
  ruangan TEXT,
  dosen_id UUID REFERENCES profiles(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. Tabel Pengajuan KRS
CREATE TABLE krs_pengajuan (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  mahasiswa_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  semester TEXT NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected')),
  catatan TEXT,
  total_sks INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 4. Tabel Detail KRS
CREATE TABLE krs_detail (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  krs_pengajuan_id UUID REFERENCES krs_pengajuan(id) ON DELETE CASCADE,
  matakuliah_id INTEGER REFERENCES matakuliah(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 5. Tabel Mata Kuliah Diambil (Riwayat)
CREATE TABLE matakuliah_diambil (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  mahasiswa_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  matakuliah_id INTEGER REFERENCES matakuliah(id) ON DELETE CASCADE,
  semester TEXT NOT NULL,
  nilai_angka DECIMAL(3,2),
  nilai_huruf TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(mahasiswa_id, matakuliah_id)
);

-- 6. Tabel Presensi Session
CREATE TABLE presensi_session (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  matakuliah_id INTEGER REFERENCES matakuliah(id) ON DELETE CASCADE,
  pertemuan INTEGER NOT NULL,
  kode_presensi TEXT,
  is_open BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 7. Tabel Presensi
CREATE TABLE presensi (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  presensi_session_id UUID REFERENCES presensi_session(id) ON DELETE CASCADE,
  mahasiswa_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  status TEXT DEFAULT 'hadir' CHECK (status IN ('hadir', 'izin', 'sakit', 'alpha')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(presensi_session_id, mahasiswa_id)
);

-- 8. Tabel Post (Materi/Tugas/Pengumuman)
CREATE TABLE post (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  matakuliah_id INTEGER REFERENCES matakuliah(id) ON DELETE CASCADE,
  author_id UUID REFERENCES profiles(id),
  judul TEXT NOT NULL,
  konten TEXT,
  jenis TEXT CHECK (jenis IN ('materi', 'tugas', 'pengumuman')),
  deadline TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 9. Tabel Submission Tugas
CREATE TABLE tugas_submission (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  post_id UUID REFERENCES post(id) ON DELETE CASCADE,
  mahasiswa_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  file_url TEXT,
  catatan TEXT,
  nilai DECIMAL(5,2),
  submitted_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(post_id, mahasiswa_id)
);

-- Indexes untuk performa
CREATE INDEX idx_matakuliah_diambil_mahasiswa ON matakuliah_diambil(mahasiswa_id);
CREATE INDEX idx_krs_pengajuan_status ON krs_pengajuan(status);
CREATE INDEX idx_presensi_mahasiswa ON presensi(mahasiswa_id);
CREATE INDEX idx_post_matakuliah ON post(matakuliah_id);
```

### Row Level Security (RLS)

```sql
-- Enable RLS
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE matakuliah ENABLE ROW LEVEL SECURITY;
ALTER TABLE krs_pengajuan ENABLE ROW LEVEL SECURITY;

-- Policies (contoh)
CREATE POLICY "Users can read own profile" ON profiles
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Anyone can read matakuliah" ON matakuliah
  FOR SELECT TO authenticated USING (true);
```
